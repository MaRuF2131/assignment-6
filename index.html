Frequently Asked Questions

1.What are the difference between var, let, and const?

  • var
    • It is function-scoped, meaning it is only limited to the function where it is declared.
    • It does not have block scope, so if declared inside a block (like an if statement), it can still be accessed outside of it.
    • It is hoisted, meaning it is moved to the top of the code before execution, but its value is undefined until assigned.
    • It can be reassigned, meaning you can change its value after declaration.
  • let
    • It is block-scoped, meaning it only exists inside the block {} where it is declared.
    • It is hoisted, but it is not initialized, so accessing it before declaration causes an error.
    • It can be reassigned, so its value can change later.
  • const
    • Like let, it is block-scoped and cannot be accessed outside its block.
    • It is hoisted, but it is not initialized, so it must be declared before use.
    • It cannot be reassigned, meaning once you set a value, you cannot change it.
    • However, if it holds an object or array, the properties inside can still be modified.

2.Wha are the difference between map(), forEach(), and filter() write?

  • map()
    • Used to transform each element in an array and create a new array.
    • It does not modify the original array; instead, it returns a new one.
    • Best when you need to modify data (e.g., doubling numbers or changing object properties).
  • forEach()
    • Used to execute a function on each element of an array without creating a new array.
    • It does not return anything (returns undefined).
    • Best when performing actions like logging values, updating UI, or modifying the original array.
  • filter()
    • Used to filter elements from an array based on a condition and create a new array.
    • Only elements that pass the condition remain in the new array.
    • Best when you need to remove unwanted elements (e.g., filtering out negative numbers or finding active users).

3.explain arrow functions and how they are different from regular functions

  • Arrow Functions (`=>`)
    • A shorter way to write functions using the `=>` syntax.
    • Do not have their own `this` keyword (they inherit `this` from the surrounding scope).
    • Cannot be used as constructors (cannot use `new`).
    • Do not have their own `arguments` object.
    • Useful for short, simple functions and callbacks.
  • Regular Functions (`function` keyword)
    • Use the `function` keyword for declaration.
    • Have their own `this` keyword, which depends on how the function is called.
    • Can be used as constructors (can use `new` to create objects).
    • Have access to the `arguments` object, which contains all passed arguments.
    • Useful when needing dynamic `this` or creating object methods.

4.how JavaScript Promises work?

  • What is a Promise?
    • A Promise represents the completion or failure of an asynchronous operation.
    • It replaces callbacks for handling async tasks.
  • States of a Promise
    • Pending: The operation is still in progress.
    • Fulfilled: The operation was successful.
    • Rejected: The operation failed.
  • How Promises Work
    • A Promise takes a function with `resolve` (success) and `reject` (failure).
    • Use `.then()` to handle success.
    • Use `.catch()` to handle errors.
    • Use `.finally()` to execute code after completion, whether success or failure.
  • Why Use Promises?
    • Avoids nested callback structures (callback hell).
    • Makes asynchronous code easier to read and manage.
    • Supports chaining multiple async operations.

5.how closures work in JavaScript?

  • What is a Closure?
    • A closure is a function that remembers the variables from its outer scope.
    • It allows access to these variables even after the outer function has executed.
  • How Closures Work
    • A function inside another function retains access to the outer function’s variables.
    • The outer function can return the inner function, keeping its variables accessible.
    • Even if the outer function is removed from the call stack, the inner function still holds those variables.
  • Why Use Closures?
    • Data Privacy: Variables inside closures cannot be accessed directly from outside.
    • Maintaining State: Useful for keeping track of values (e.g., counters, configurations).
    • Event Handlers & Callbacks: Helps retain context in asynchronous code.
